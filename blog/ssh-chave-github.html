Adicionando uma Chave SSH ao GitHub

Como obter e configurar suas chaves SSH do Git e do GitHub. Verificar a existência de uma chave SSH.

Sobre chaves SSH...

Você pode usar o SSH para executar operações Git em repositórios em GitHub.com. Para obter mais informações, confira "Sobre o SSH".
Se você tiver uma chave SSH existente, poderá usar a chave para autenticar as operações do Git por SSH.

Verificar se há chaves SSH

Antes de gerar uma nova chave SSH, verifique se há chaves existentes no computador local.

$ ls -al ~/.ssh

Verifique a listagem do diretório para verificar se você já tem uma chave SSH pública. Por padrão, os nomes de arquivos de chaves públicas com suporte para o GitHub são um dos seguintes.

  • id_rsa.pub
  • id_ecdsa.pub
  • id_ed25519.pub

Se já existem chaves, pule para a seção Copiar a sua chave SSH pública.
Se não houver nenhum resultado ou se o diretório ainda não existir, você não terá um par de chaves SSH existente no local padrão. Você pode criar um novo par de chaves SSH na próxima etapa.

Gerando uma nova chaves SSH

Você pode gerar uma nova chave SSH no computador local. Depois de gerar a chave, você pode adicionar a chave pública à sua conta em Github.com para habilitar a autenticação para operações do Git no SSH.
1. Abra um Terminal.
2. Execute o comando abaixo, substituindo o email usado no exemplo pelo seu endereço de email do GitHub.

# O comando irá criar uma pasta .ssh na pasta Home do usuário
$ mkdir $HOME/.ssh
# Em seguinda, geramos um novo conjunto de chaves
$ ssh-keygen -t ed25519 -C "your_email@example.com"

Observação: se estiver usando um sistema herdado que não dá suporte ao algoritmo Ed25519, use:

$ ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

Quando for solicitado a inserir um arquivo para salvar a chave, pressione Enter para aceitar o local padrão do arquivo. Observe que, se você criou chaves SSH anteriormente, ssh-keygen pode pedir que você reescreva outra chave. Nesse caso, recomenda-se criar uma chave SSH personalizada. Para fazer isso, digite o local do arquivo padrão e substitua id_ALGORITHM pelo nome da chave personalizada.

> Enter a file in which to save the key (/home/YOU/.ssh/id_ALGORITHM):[Press enter]

No prompt, digite uma frase secreta segura. Se não quiser a chave com uma frase secreta, apenas pressione Enter.

> Enter passphrase (empty for no passphrase): [Type a passphrase] > Enter same passphrase again: [Type a passphrase again]

Adicionar sua chave SSH ao ssh-agent

Antes de adicionar uma nova chave SSH ao agente para gerenciar suas chaves, você deve verificar as chaves SSH existentes e gerado uma nova chave SSH.

1 Inicie o ssh-agent em segundo plano.

$ eval "$(ssh-agent -s)" > Agent pid 59566

Dependendo do seu ambiente, talvez seja necessário usar um comando diferente. Por exemplo, talvez seja necessário usar o acesso raiz executando sudo -s -H antes de iniciar o ssh-agent ou usar exec ssh-agent bash ou exec ssh-agent zsh para executar o ssh-agent.

2 Adicione sua chave SSH privada ao ssh-agent.

Se você criou sua chave com um nome diferente ou está adicionando uma chave existente que tenha outro nome, substitua id_ed25519 no comando pelo nome do arquivo de chave privada.

$ ssh-add ~/.ssh/id_ed25519

Como adicionar uma nova chave SSH à sua conta

1 Copie a chave pública SSH para a sua área de transferência.

Se o seu arquivo de chave pública SSH tiver um nome diferente do código de exemplo, modifique o nome do arquivo para corresponder à sua configuração atual. Ao copiar sua chave, não adicione novas linhas nem espaços em branco.

$ cat ~/.ssh/id_ed25519.pub # Selecione e copie o conteúdo do arquivo id_25519.pub

2 Settings

No canto superior direito de qualquer página, clique na foto do seu perfil e em Settings.

3 Access

Na seção "Acesso" da barra lateral, clique em:
SSH and GPG Keys

4 New SSH Key

Clique em New SSH Key ou Add SSH Key.

5 Title

No campo "Title" (Título), adicione uma etiqueta descritiva para a nova chave. Por exemplo, se estiver usando um pc pessoal, você poderá chamar essa chave de "PC Pessoal".

5 Key type

Selecione o tipo de chave: autenticação ou assinatura. Para saber mais sobre a assinatura de commit, confira "Sobre a verificação de assinatura de commit".

6 Key

No campo "Chave", cole sua chave pública.

8 Clique em Add SSH Key.

9 Se solicitado, confirme acesso à sua conta em GitHub. Para obter mais informações, confira "Modo sudo".

Finalmente, teste a sua autenticação com:

$ ssh -T git@github.com

Se você seguiu todas as etapas corretamente, você deve ver esta mensagem:

Hi your_user_name! You've successfully authenticated, but GitHub does not provide shell access.
